# 광동제약 방문예약 시스템 - 기본 기능 테스트 계획

## 애플리케이션 개요

광동제약 방문예약 시스템은 React + TypeScript 기반의 웹 애플리케이션으로, 방문 예약 신청 및 관리를 위한 종합 시스템입니다.

주요 기능:
- **방문 예약 신청**: 방문자 정보 입력 및 예약 신청
- **예약 진행 상황 조회**: 예약 상태 확인 및 진행 상황 추적
- **관리자 승인/반려**: 관리자의 예약 승인 및 반려 처리
- **임직원 체크인**: 방문자 체크인 처리
- **공지사항 및 FAQ**: 공지사항 및 자주 묻는 질문 조회

## 테스트 시나리오

### 1. 홈페이지 기본 기능

**Seed:** `tests/seed.spec.ts`

#### 1.1 홈페이지 로딩

**단계:**
1. 홈페이지(`/`)로 이동
2. 페이지 로딩 완료 대기

**예상 결과:**
- 페이지가 정상적으로 로드됨
- 헤더가 표시됨
- 메인 콘텐츠 영역이 표시됨
- 푸터가 표시됨

#### 1.2 공장 선택 기능

**단계:**
1. 홈페이지에서 공장 선택 드롭다운 찾기
2. GMP공장 또는 식품공장 선택

**예상 결과:**
- 공장 선택 옵션이 표시됨
- 선택한 공장이 반영됨

### 2. 방문 예약 프로세스

#### 2.1 예약 동의 페이지

**단계:**
1. 홈페이지에서 예약 시작 버튼 클릭
2. 동의 페이지(`/reservation/agreement`)로 이동
3. 약관 동의 체크박스 확인

**예상 결과:**
- 동의 페이지가 표시됨
- 약관 내용이 표시됨
- 동의 체크박스가 활성화됨

#### 2.2 예약 폼 입력

**단계:**
1. 동의 후 다음 단계로 진행
2. 예약 폼(`/reservation/form`)에서 정보 입력
   - 방문자 정보 입력
   - 방문 목적 선택
   - 방문 일시 선택

**예상 결과:**
- 폼 필드가 정상적으로 표시됨
- 입력값이 저장됨
- 유효성 검사가 작동함

### 3. 예약 조회 기능

#### 3.1 진행 상황 조회

**단계:**
1. 진행 상황 페이지(`/progress`)로 이동
2. 예약 번호 또는 QR 코드로 조회

**예상 결과:**
- 예약 정보가 표시됨
- 현재 진행 상태가 표시됨
- 다음 단계 안내가 표시됨

### 4. 관리자 기능

#### 4.1 관리자 승인 페이지

**단계:**
1. 관리자 승인 페이지(`/admin/approval`)로 이동
2. 대기 중인 예약 목록 확인
3. 승인 또는 반려 처리

**예상 결과:**
- 예약 목록이 표시됨
- 승인/반려 버튼이 작동함
- 상태 변경이 반영됨

### 5. 임직원 기능

#### 5.1 임직원 대시보드

**단계:**
1. 임직원 대시보드(`/employee`)로 이동
2. 체크인 탭 확인
3. 방문자 체크인 처리

**예상 결과:**
- 대시보드가 표시됨
- 탭 전환이 작동함
- 체크인 기능이 작동함

### 6. 공지사항 및 FAQ

#### 6.1 공지사항 페이지

**단계:**
1. 공지사항 페이지(`/notice`)로 이동
2. 공지사항 목록 확인

**예상 결과:**
- 공지사항 목록이 표시됨
- 공지사항 상세 내용 확인 가능

#### 6.2 FAQ 페이지

**단계:**
1. FAQ 페이지(`/faq`)로 이동
2. FAQ 목록 확인
3. FAQ 항목 클릭하여 답변 확인

**예상 결과:**
- FAQ 목록이 표시됨
- 아코디언 또는 드롭다운으로 답변 표시
- 답변이 정상적으로 표시됨
