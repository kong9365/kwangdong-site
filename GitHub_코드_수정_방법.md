# GitHub에서 코드 직접 수정하는 방법

## ✅ 네, 가능합니다!

GitHub는 웹 브라우저에서 직접 코드를 수정할 수 있는 기능을 제공합니다.

---

## 📝 방법 1: GitHub 웹 에디터 사용 (간단)

### 단계별 가이드

#### 1단계: 파일 선택
1. GitHub 저장소 페이지 접속
   - 예: `https://github.com/kong9365/kwangdong-site`
2. 수정하고 싶은 파일 클릭
   - 예: `README.md`, `src/App.tsx` 등

#### 2단계: 편집 모드 진입
1. 파일 내용이 표시된 페이지에서
2. **연필 아이콘 (✏️ Edit this file)** 클릭
   - 오른쪽 상단에 위치

#### 3단계: 코드 수정
1. 웹 에디터에서 코드 직접 수정
2. 실시간으로 변경사항 확인 가능

#### 4단계: 변경사항 저장
1. 페이지 하단의 **"Commit changes"** 섹션으로 스크롤
2. **Commit message** 입력
   - 예: "README 파일 업데이트"
   - 예: "버그 수정"
3. **"Commit changes"** 버튼 클릭

#### 5단계: 완료!
- 변경사항이 저장소에 바로 반영됩니다
- Vercel이 자동으로 감지하여 재배포합니다

---

## 💻 방법 2: GitHub Codespaces 사용 (고급)

### Codespaces란?
- GitHub에서 제공하는 클라우드 개발 환경
- VS Code와 유사한 인터페이스
- 터미널, 디버깅 등 모든 개발 도구 사용 가능

### 사용 방법
1. 저장소 페이지에서 **"Code"** 버튼 클릭
2. **"Codespaces"** 탭 선택
3. **"Create codespace on main"** 클릭
4. 브라우저에서 VS Code 같은 에디터 열림
5. 코드 수정 후 Git으로 커밋 및 푸시

---

## 🔄 수정 후 자동 배포

### Vercel 자동 배포
```
GitHub 코드 수정
    ↓
자동으로 커밋
    ↓
Vercel이 변경사항 감지
    ↓
자동으로 재배포
    ↓
웹사이트 업데이트 완료!
```

**시간**: 약 1-2분 소요

## 📋 실제 예시

### 예시 1: README 파일 수정

1. `https://github.com/kong9365/kwangdong-site` 접속
2. `README.md` 파일 클릭
3. 연필 아이콘 클릭
4. 내용 수정
5. "Update README" 메시지 입력
6. "Commit changes" 클릭
7. 완료!

### 예시 2: 코드 파일 수정

1. `src/App.tsx` 파일 클릭
2. 연필 아이콘 클릭
3. 코드 수정
   ```typescript
   // 수정 전
   return <div>Hello</div>;
   
   // 수정 후
   return <div>Hello World</div>;
   ```
4. "Fix typo" 메시지 입력
5. "Commit changes" 클릭
6. 완료!

## ⚠️ 주의사항

### 1. 파일 생성도 가능
- **"Add file"** → **"Create new file"** 클릭
- 새 파일 생성 및 코드 작성 가능

### 2. 파일 삭제도 가능
- 파일 페이지에서 **휴지통 아이콘** 클릭
- 삭제 후 커밋

### 3. 브랜치 생성
- 수정 시 **"Create a new branch"** 선택 가능
- 메인 브랜치에 직접 커밋하지 않고 새 브랜치 생성
- 나중에 Pull Request로 병합

### 4. 파일 크기 제한
- 매우 큰 파일은 웹 에디터에서 수정 어려울 수 있음
- 이 경우 로컬에서 수정 후 푸시 권장

---

## 🆚 로컬 수정 vs 웹 수정

| 항목 | 로컬 수정 | 웹 수정 |
|------|----------|---------|
| **편의성** | 개발 도구 필요 | 브라우저만 있으면 됨 |
| **속도** | 빠름 | 중간 |
| **기능** | 모든 기능 사용 가능 | 기본 편집 기능만 |
| **적합한 경우** | 대규모 수정 | 간단한 수정 |
| **인터넷** | 필요 없음 | 필요 |

## 📚 추가 팁

### 파일 히스토리 보기
- 파일 페이지에서 **"History"** 클릭
- 모든 변경 이력 확인 가능

### 변경사항 비교
- 커밋 메시지 클릭
- 변경 전/후 코드 비교 가능

### 되돌리기
- 이전 버전으로 되돌리기 가능
- "Revert" 버튼 사용
