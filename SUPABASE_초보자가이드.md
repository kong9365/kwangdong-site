# Supabase 초보자 가이드 📚

## 🤔 Supabase가 뭔가요?

**Supabase**는 웹사이트나 앱에서 필요한 **데이터베이스**를 쉽게 사용할 수 있게 해주는 서비스입니다.

### 쉽게 비유하면:
- **일반 데이터베이스**: 복잡한 서버 설치와 관리 필요 (어려움 😰)
- **Supabase**: 클라우드에서 제공하는 데이터베이스 (쉬움 😊)

## 💡 우리 프로젝트에서 Supabase를 왜 사용하나요?

### 1. **데이터 저장소 역할** 📦

웹사이트에서 입력한 정보를 어딘가에 저장해야 합니다:

```
사용자가 입력 → Supabase에 저장 → 나중에 다시 불러오기
```

**예시:**
- 방문 예약 신청 → Supabase에 저장
- 예약번호로 검색 → Supabase에서 찾아서 보여주기
- 공지사항 작성 → Supabase에 저장 → 다른 사람들이 볼 수 있게

### 2. **실시간 데이터 동기화** 🔄

여러 사람이 동시에 사용해도:
- 한 사람이 예약하면 → 다른 사람도 바로 확인 가능
- 관리자가 승인하면 → 사용자에게 바로 반영

### 3. **보안 관리** 🔒

- 중요한 정보를 안전하게 보관
- 누가 무엇을 볼 수 있는지 제어 (RLS 정책)

## 🏗️ 우리 프로젝트의 구조

```
┌─────────────────┐
│   웹사이트      │  ← 사용자가 보는 화면
│  (React 앱)     │
└────────┬────────┘
         │
         │ 데이터 요청/전송
         │
┌────────▼────────┐
│   Supabase     │  ← 데이터 저장소
│   (데이터베이스) │
└─────────────────┘
```

## 📊 우리가 저장하는 데이터 종류

### 1. **방문 예약 정보** (`visit_requests`)
- 예약번호
- 방문 날짜
- 방문 목적
- 승인 상태 등

### 2. **방문자 정보** (`visitor_info`)
- 방문자 이름
- 연락처
- 차량번호 등

### 3. **공지사항** (`notices`)
- 제목
- 내용
- 작성일 등

### 4. **FAQ** (`faqs`)
- 질문
- 답변
- 카테고리 등

### 5. **문자 알림 로그** (`sms_notifications`)
- 전송한 문자 내용
- 전송 상태
- 전송 시간 등

## 🔧 Supabase의 주요 기능

### 1. **데이터베이스 (Database)**
- 테이블 형태로 데이터 저장
- SQL 쿼리로 데이터 조회/수정/삭제

### 2. **인증 (Authentication)**
- 사용자 로그인/회원가입 관리
- (우리 프로젝트에서는 아직 사용 안 함)

### 3. **실시간 (Realtime)**
- 데이터 변경 시 자동 업데이트
- (우리 프로젝트에서는 아직 사용 안 함)

### 4. **Storage**
- 파일 업로드/저장
- (우리 프로젝트에서는 아직 사용 안 함)

## 🎯 왜 Supabase를 선택했나요?

### 장점:
1. ✅ **무료로 시작 가능** (개인 프로젝트용)
2. ✅ **설정이 간단함** (서버 설치 불필요)
3. ✅ **자동 백업** (데이터 안전)
4. ✅ **빠른 속도** (클라우드 서버)
5. ✅ **보안 기능 내장** (RLS 정책)

### 다른 선택지와 비교:
- **직접 서버 구축**: 어렵고 비용 큼 ❌
- **Firebase**: Google 서비스, 비슷하지만 Supabase가 더 SQL 친화적
- **MySQL 직접 설치**: 복잡하고 관리 어려움 ❌

## 📝 실제 사용 예시

### 예시 1: 방문 예약 신청
```
1. 사용자가 웹사이트에서 예약 정보 입력
2. "신청하기" 버튼 클릭
3. 웹사이트가 Supabase에 데이터 전송
4. Supabase가 데이터 저장
5. 예약번호 생성해서 사용자에게 보여줌
```

### 예시 2: 예약현황 조회
```
1. 사용자가 예약번호 입력
2. 웹사이트가 Supabase에 "이 예약번호 찾아줘" 요청
3. Supabase가 데이터 찾아서 전달
4. 웹사이트가 화면에 표시
```

## 🔐 보안은 어떻게 되나요?

### RLS (Row Level Security) 정책
- **공지사항, FAQ**: 모든 사람이 읽기 가능 ✅
- **방문 요청**: 본인 것만 볼 수 있게 설정 가능
- **관리자 기능**: 관리자만 승인/반려 가능

## 💰 비용은 얼마인가요?

- **무료 플랜**: 
  - 데이터베이스 500MB
  - 월 2GB 대역폭
  - 개인 프로젝트에 충분함!

- **유료 플랜**: 
  - 더 많은 용량 필요 시
  - 우리 프로젝트는 아직 무료로 충분함

## 🎓 초보자를 위한 핵심 정리

### Supabase = 클라우드 데이터베이스 서비스

**역할:**
- 웹사이트의 데이터 저장소
- 정보를 안전하게 보관
- 필요할 때 빠르게 가져오기

**우리 프로젝트에서:**
- 방문 예약 정보 저장
- 공지사항 저장
- FAQ 저장
- 모든 데이터의 중심 저장소

**비유:**
- 웹사이트 = 가게
- Supabase = 창고
- 데이터 = 상품

가게(웹사이트)에서 상품(데이터)을 보여주고, 창고(Supabase)에 저장하는 것!
