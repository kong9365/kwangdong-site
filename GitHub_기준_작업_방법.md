# GitHub 최신 코드 기준 작업 방법

## 🎯 작업 프로세스

```
1. GitHub에서 최신 코드 확인
    ↓
2. 최신 코드를 기준으로 수정
    ↓
3. 수정된 코드를 GitHub에 푸시
    ↓
4. Vercel 자동 배포
```

---

## 📋 제가 항상 하는 작업 순서

### 1단계: GitHub 최신 코드 확인
```javascript
// GitHub에서 최신 파일 내용 가져오기
mcp_github_get_file_contents({
  owner: "kong9365",
  repo: "kwangdong-site",
  path: "수정할_파일_경로"
})
```

### 2단계: 최신 코드 분석
- GitHub에 저장된 현재 코드 확인
- 수정할 부분 파악
- 기존 코드 구조 유지

### 3단계: 코드 수정
- 최신 코드를 기준으로 수정
- 기존 기능 유지하면서 변경
- 테스트 가능한 코드 작성

### 4단계: GitHub에 푸시
```javascript
// 수정된 코드를 GitHub에 업로드
mcp_github_create_or_update_file({
  owner: "kong9365",
  repo: "kwangdong-site",
  path: "수정한_파일_경로",
  content: "수정된_코드",
  message: "수정 내용 설명",
  branch: "main"
})
```

### 5단계: 자동 배포
- Vercel이 자동으로 감지
- 1-2분 후 웹사이트 업데이트

---

## ✅ 항상 보장하는 것

### 1. 최신 코드 기준
- ✅ GitHub의 최신 코드를 먼저 확인
- ✅ 로컬 파일이 아닌 GitHub 코드 기준
- ✅ 다른 사람이 수정한 내용도 반영

### 2. 안전한 수정
- ✅ 기존 코드 구조 유지
- ✅ 다른 기능에 영향 없도록 수정
- ✅ 코드 검증 후 푸시

### 3. 추적 가능
- ✅ 모든 변경사항이 GitHub에 기록
- ✅ 언제든지 이전 버전으로 되돌리기 가능
- ✅ 변경 이력 확인 가능

---

## 🔄 실제 작업 예시

### 예시: 홈페이지 제목 수정

#### 1단계: GitHub에서 최신 코드 확인
```
제가: GitHub에서 src/pages/Home.tsx 파일 가져오기
→ 현재 코드 확인
→ "광동제약 방문예약" 텍스트 위치 파악
```

#### 2단계: 코드 수정
```
제가: 
- 기존: "광동제약 방문예약"
- 수정: "광동제약 방문 신청"
- 나머지 코드는 그대로 유지
```

#### 3단계: GitHub에 푸시
```
제가: 수정된 코드를 GitHub에 업로드
→ 커밋 메시지: "홈페이지 제목 변경"
```

#### 4단계: 자동 배포
```
Vercel: 변경사항 감지 → 자동 재배포
→ 1-2분 후 웹사이트 업데이트
```

## 💡 장점

### ✅ 항상 최신 상태
- GitHub의 최신 코드 기준으로 작업
- 다른 사람이 수정한 내용도 반영
- 코드 충돌 방지

### ✅ 안전한 수정
- 기존 코드 구조 유지
- 다른 기능에 영향 없음
- 테스트 가능한 코드

### ✅ 추적 가능
- 모든 변경사항 기록
- 언제든지 되돌리기 가능
- 변경 이력 확인

---

## 📝 작업 시 항상 확인하는 것

### 1. 파일 존재 여부
- GitHub에 파일이 있는지 확인
- 없으면 새로 생성

### 2. 최신 내용
- GitHub의 최신 버전 확인
- SHA 값으로 버전 확인

### 3. 코드 구조
- 기존 코드 구조 파악
- 일관성 유지

### 4. 의존성
- 다른 파일과의 관계 확인
- 영향 범위 파악

---

## 🎯 핵심 정리

1. **항상 GitHub 기준** - 최신 코드를 먼저 확인
2. **안전한 수정** - 기존 구조 유지하면서 변경
3. **자동 푸시** - 수정 후 바로 GitHub에 업로드
4. **자동 배포** - Vercel이 자동으로 재배포
